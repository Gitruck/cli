## Context

D-P0 把 vocab 词表配置化,style 块留空占位(其早期骨架 `{palette, aspect_ratio}` 已随本 change 的对抗审查回写修订——palette/aspect_ratio 本身就是框架预设的两个视觉维度)。曾探索过 style 8 子块 schema,被主理人元洞察否决:从《实在界漫游指南》的 skill 归纳维度,归纳本身就是路径依赖。

对四个栏目 skill 的六路探底给出经验底座:每个 skill 都可四分为 **v(栏目取值)/ k(该栏目自己长出的组织骨架)/ 工程契约 / 方法论**——其中只有工程契约属于框架,且实测契约文档并非天然干净(gsap-emit-pattern.md 正文夹带 #BD2B35、Noto 字体、「旁白主导=纯黑」等栏目取值,收录必须**中性化改写**)。反面教材:叙事功能枚举 ≥6 处重复定义且 7/8 口径打架——**没有栏目内共享词表,家族必漂移**。

生态载体事实:skill 四条注册通道中只有 `~/.claude/skills` 可程序化闭环写入;「skill 生成 skill」已有先例(lark-skill-maker/skill-creator);cli 仓已有 skill 分发范式(`cli/skills/` 随 npm + `gtrk skills install`)。

## Goals / Non-Goals

**Goals:**
- style 块定形为框架零解析的引用清单,风格知识全部活在用户 skill 里(可执行/黑盒/本地)。
- 交付 meta skill `gtrk-style-maker`:引导任何栏目主定义**他自己的** k 和 v,零视觉维度预设,且零预设是**可审计**的(不靠措辞自觉)。
- 自有管线契约经中性化改写收进 `cli/contracts/`,版本化引用 + handoff→契约映射表。
- 黑盒脱敏与零预设「结构化 + 验收化」:分发物整体 grep 自证,不靠逐次人工检查。

**Non-Goals:**
- 不定义任何视觉维度、不内置任何风格取值(连「有 narrative」「视觉分 MG/AI」都不假设)。
- 不做 lane/handoff 注册表化(P2 承重面):本 change 对注册集只引用不复述、不扩集。
- 不做《实在界》种子重构本身(P1b,作坊侧,另有计划文档;本 change 的种子/重生成素材**以 P1b 收口后为基准**)。
- 不做 L1/L3 对 style 的承载、不做 bundle 上云实现(只留 `bundle_ref` 占位)。
- 不做 `gtrk column` 登记类子命令(meta skill 直接读写本地 JSON;写路径以 CLI paths 口径为准,skill 文档不写死路径)。
- 管线文档生产者(splitter 类,产出的是 handoff 文档本身而非某 lane 产物)的登记语义留待 P2 与注册表一起定,本期不给其登记 `produces`。

## Decisions

- **D1 style = 不透明引用清单,而非结构化风格 schema**
  ```
  style: {
    skills:  [ { id, ref, produces?, status?, routing? } ],   // produces: string|string[];status/role 框架不消费仅透传
    shared:  [ { id, ref, role? } ],
    bundle_ref?: string                                        // 上云占位:config→DB 与 bundle→OSS 分家
  }
  ```
  框架对 `ref` 内容 MUST NOT 解析。条目最小合法字段 = id+ref,非法条目跳过+提示不抛(字段级校验,不违反零解析)。8 子块**降格**为 meta skill 的卡壳追问语料(受 D4 的操作化约束)。备选「结构化 schema + 自由扩展字段」被否:只要框架读得懂风格字段,就会有代码依赖它,预设就焊死了——palette/aspect_ratio 的废弃即此判据的第一次执行。
- **D2 `produces` 惰性路由:命中才生效,未命中静默**
  `produces` 登记产物绑定的 handoff 类型,命中 **handoff 注册集**(取值与演进归 lane/handoff 侧管理,本工件不复述枚举;当前注册集=现行四车道,是 P2 注册表化前的过渡事实)→ 可参与派单路由;未命中 → **静默**惰性登记(异构栏目的 produces 天然 100% 未命中,逐条提示等于框架对自称零预设的栏目唠叨)。仅「与注册值大小写不同或编辑距离 ≤2」提示疑似拼写(每次 resolve 每值至多一条);`routing:"none"` 显式豁免。备选「未命中一律提示」被否:提示收益只惠及 lane 形状的栏目,正是「名义零预设、实际实在界式才好用」。
- **D3 meta skill 载体与产物落点**
  meta skill 进 `cli/skills/gtrk-style-maker` + skill 分发清单,`gtrk skills install` 安装。产出的用户 skill 落 `~/.claude/skills`;登记写入本地栏目配置(位置以 `paths.ts` 口径为准,skill 文档写「以 gtrk paths 为准(当前 ~/.gitruck/columns/)」,不做第二个路径权威源),同层写入为**追加**语义。**L0 不内置 style 清单**——默认栏目《实在界》的家族也走 L2 登记,否则在默认栏目上追加 skill 须 fork 整块,配置层结构性复现副本漂移。「正本→安装」单向流写进产出的 skill README。
- **D4 启发式引导:问证据、不问维度(七招,操作化)**
  七招:①作品逆向(不适用分支:从零新栏目无作品→借他人作品定位「最像我想要的」);②禁令先行;③从工序提决策树;④铁律化自检;⑤词表沉淀;⑥家族切分(不适用分支:单产线栏目不切家族);⑦exemplar 优先(不适用分支:实拍混剪类产线以「选段判例」代成品件)。卡壳追问操作化三约束:单次访谈 ≤3 条;每条必须由用户已说内容触发并记录触发语;禁按预设语系逐项顺次遍历。**为什么维度归纳致命而工艺归纳可接受**:维度归纳把「栏目世界观」写进框架(约束内容),工艺归纳只约束「引导过程」且带证伪出口——异构试金石失败即证明工艺有预设残留,须回改工艺;spec 的可审计判据(维度首提者=用户、预设语系 grep)把这条边界变成验收而非自觉。
- **D5 种子机制:脱敏削减 + 双道等强验收**
  种子只示范**组织维度 + 词表 + 自检结构**,不含平台 prompt 内容;外部供应商/平台名以中性占位替换(正本在用户作坊)——这是「越真实的平台向种子越黑盒不兼容」的显式取舍。种子与重生成素材一律以 **P1b 收口后正本**为基准(否则旧版的 60% 留白铁律/7-8 口径 bug/红点不稀缺会被验收基准固化,与主理人裁决对撞)。双道验收等强:重生成道(等价收敛)与异构道(可审计零预设 + 自检全过 + 落盘完整),防止优化压力单边朝实在界。
- **D6 契约库:中性化改写 + `cli/contracts/` + 封闭列举 + 映射表**
  契约正本放 `cli/contracts/`(管线资产归管线位置;放 meta skill 目录下会让不经 meta skill 的用户和渲染管线自身都绕道,且 skill 改名即断路径),package.json files 增列随 npm 分发。GSAP-emit 收录时**中性化改写**:只保留机器可判定属性(template 包裹/paused 注册/确定性/自包含/依赖可达/字面值禁 var);示例色值字体一律中性占位;「旁白主导=纯黑/叠加=透明」改写为「根底色透明与否必须显式声明,具体色值由调用方按栏目规则指定」;1920×1080 标注为「当前引擎版本约束(参数化预留)」;「全片最多一个红点」等审美铁律**不入库**(留在栏目 skill 侧)。收录范围封闭列举(渲染引擎合规/.gtrk 契约/派单格式),扩项走 change。契约库维护 handoff→契约映射表(允许「暂无契约」),meta skill 查表不硬编码。
- **D7 折叠算子沿用 D-P0:style 整块 OVERRIDE**
  「换装」语义,数组不逐条 merge;配合 D3 的「L0 无清单」,同层追加 + 层间换装两个语义互不打架。

## Risks / Trade-offs

- [meta skill 产出质量依赖访谈发挥] → 双道等强验收 + 七招硬工序 + 每招正反例与不适用分支。
- [中性化改写可能割掉契约文档里对创作者有用的实感示例] → 示例保留但取值参数化(「此处填你栏目的底色」);实在界的字面值版本留在栏目 skill 侧作为参照。
- [契约剥离后既有 skill 引用断档] → P1b 收口时切换引用;过渡期正本唯一在 `cli/contracts/`,旧位置加指向头注(作坊侧动作)。
- [重生成道仍可能诱导把实在界结构泄进提示词] → 异构道判据等强(维度首提审计 + 预设语系 grep),泄漏在异构道必然显形。
- [框架零解析意味着框架无法替用户保证风格质量] → 接受。质量抓手是 meta skill 的工艺,不是运行时校验——白标「责权利甩给客户」的题中之义。

## Migration Plan
